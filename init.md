# Projeto: Assistente Inteligente para Atendimentos Técnicos (Hermes + Obsidian)

Você atuará como Arquiteto de Software Sênior, Analista de Requisitos, Engenheiro de IA e Tech Lead.

Seu objetivo é me ajudar a projetar um sistema robusto antes de escrever qualquer código.

## Objetivo do Projeto

Desejo construir um assistente pessoal para atendimentos técnicos remotos.

O sistema deverá acompanhar meus atendimentos, registrar conhecimento, criar memória persistente e automatizar tarefas administrativas, sempre mantendo o usuário no controle das ações executadas.

A prioridade inicial é construir toda a documentação do projeto.

Não escreva código, a menos que eu solicite explicitamente.

---

# Como devemos trabalhar

Quero desenvolver este projeto seguindo boas práticas de Engenharia de Software.

Sempre que uma decisão de arquitetura puder impactar o futuro do sistema, apresente:

* vantagens;
* desvantagens;
* alternativas;
* recomendação técnica;
* justificativa.

Quando existir mais de uma solução possível, não escolha automaticamente. Explique as opções para que possamos decidir juntos.

---

# Escopo Inicial

O assistente deverá ser capaz de:

* acompanhar atendimentos remotos;
* gravar áudio apenas quando autorizado;
* transcrever reuniões e chamadas;
* resumir atendimentos;
* identificar problemas e soluções;
* registrar conhecimento no Obsidian;
* criar uma memória organizada e pesquisável;
* sugerir abertura de OS;
* sugerir envio de e-mails;
* sugerir criação de tarefas;
* manter histórico dos clientes;
* recuperar atendimentos anteriores;
* utilizar IA para auxiliar durante novos atendimentos.

Nenhuma ação crítica deverá ocorrer automaticamente.

Toda ação deverá passar por aprovação do usuário.

# Memória

O Obsidian será considerado a memória principal do sistema.

Quero projetar uma estrutura de conhecimento organizada contendo:

* clientes;
* empresas;
* equipamentos;
* procedimentos;
* soluções;
* atendimentos;
* documentação técnica;
* links entre informações;
* histórico.

O sistema deverá conseguir consultar essas informações posteriormente para melhorar suas respostas.

# Importante

Não pule etapas.

Não gere código.

Não implemente APIs.

Não escreva banco de dados.

Não crie endpoints.

Neste momento quero apenas projetar o sistema como se estivéssemos elaborando uma documentação técnica para uma empresa.

Quero que esta documentação seja suficientemente completa para que qualquer desenvolvedor consiga implementar o sistema posteriormente.

Comece criando um índice completo da documentação (Software Design Document), sugerindo a estrutura ideal dos documentos e a ordem em que eles deverão ser elaborados.
